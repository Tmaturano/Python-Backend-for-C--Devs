# Lesson 4 - Classes, Dataclasses and OOP

## Overview

This lesson covers object-oriented programming in Python. While you're familiar with OOP from C#, Python's approach has unique features and conventions that differ from the .NET world.

## Topics Covered

- **Classes**: Defining classes, instance variables, and methods
- **Inheritance**: Single and multiple inheritance patterns
- **Interfaces/Protocols**: Abstract classes and interface definitions
- **Dataclasses**: Modern Python approach to simple data-holding classes
- **OOP Best Practices**: Pythonic design patterns and conventions

## Files

- `classes.py` - Class definition and usage
- `inheritance.py` - Inheritance hierarchies and method resolution
- `abstract_class.py` - Abstract base classes and enforcing interfaces
- `interfaces.py` - Protocol definitions and interface patterns
- `dataclasses_new.py` - Using the modern `@dataclass` decorator

## Key Takeaway

Python's OOP model is flexible and supports multiple inheritance, but with different conventions than C#. Dataclasses provide a cleaner alternative for simple data classes compared to writing boilerplate code.
